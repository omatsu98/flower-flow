# Flower Flow

地域別に店舗の花在庫、納品、売上を共有管理するアプリです。

## ローカルで起動

```sh
npm start
```

`npm` がない環境では次でも起動できます。

```sh
node server.js
```

起動後、同じパソコンでは次を開きます。

```text
http://127.0.0.1:4173/
```

同じWi-Fi内の別端末から使う場合は、このパソコンのローカルIPアドレスを使って開きます。

```text
http://パソコンのIPアドレス:4173/
```

## 共有保存

ブラウザで直接 `index.html` を開いた場合は、その端末だけに保存されます。

`npm start` で起動したURLから開くと、全員が同じサーバー保存データを使います。保存データは次のファイルに残ります。

```text
data/flower-flow-state.json
```

在庫、返品、納品、売上、店舗、地域、花材を変更すると自動保存されます。保存状況は画面上部に表示されます。

## 画像から在庫項目・売上を読み取る

画像読取を使うサーバーでは、OpenAI APIキーを環境変数に設定します。APIキーはブラウザには保存せず、サーバーだけに設定してください。

```sh
export OPENAI_API_KEY="your-api-key"
node server.js
```

必要に応じて画像読取モデルを変更できます。未設定時は `gpt-5.4-mini` を使用します。

```sh
export OPENAI_VISION_MODEL="gpt-5.4-mini"
```

画像取込欄では「在庫表」と「売上表」を切り替えられます。

- 在庫表: 最大4枚の画像から、店舗名、左列の花材名、単価を読み取ります。読取結果は修正してから追加でき、同じ日付・店舗・花材・単価の項目は重複追加されません。
- 売上表: 店舗名、売上金額、販売数量、日付を読み取ります。同じ日・同じ店舗の画像売上は上書きされ、二重加算されません。

画像内で `し` または `シ` と略されている花材は `樒`、`び` または `ビ` と略されている花材は `ビシャコ` として展開します。

## Safariやブラウザに保存

公開URL、またはローカルサーバーURLをSafariで開き、共有ボタンから「ホーム画面に追加」を選ぶとアプリのように保存できます。

## Renderへ公開

このフォルダにはRender用の `render.yaml` が入っています。GitHubへ登録してRenderのBlueprintから公開すると、シンガポール地域のWebサービスと1GBの永続保存領域が作成されます。

Render側では `OPENAI_API_KEY` だけを秘密の環境変数として設定してください。画像読取を使わない場合は未設定でも、在庫管理と共有保存は利用できます。

永続保存領域を使うため、設定は有料のStarterプランです。無料プランへ変更すると、再起動や再公開の際に保存データが消える可能性があります。

## その他のサーバーへ公開

Node.jsが動くサーバーへ、このフォルダ一式をアップロードして次を実行します。

```sh
npm start
```

または次を実行します。

```sh
node server.js
```

公開先の管理画面で設定する起動コマンドは次です。

```text
npm start
```

公開先の管理画面で設定するポートは、多くの場合は自動で `PORT` という環境変数から渡されます。このアプリはその設定に対応しています。

長く使う場合は、`data/flower-flow-state.json` が消えない保存領域に置かれる公開先を選んでください。
